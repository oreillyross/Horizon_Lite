Building the Intelligence Architecture

Before any data flows in, analysts construct a three-level prediction hierarchy. This is deliberate — Horizon is a thinking tool: you declare what you're watching for, then let the system tell you if it's happening.

Step-by-step setup
  1. 1

    Create a Theme

    A Theme is the top-level named domain of concern — for example "Russian Information Operations in the EU" or "Supply Chain Disruption Risk". It holds all subsequent analysis and is the unit that gets a synopsis. Up to ~20 scenarios can nest under a single theme. The theme name must be unique across the system.

  2. 2

    Attach Scenarios to the Theme

    Each Theme contains one or more Scenarios — discrete, named futures or outcomes the analyst is tracking. A Scenario is scoped to an Analyst Group (team-level access control) and linked to its parent Theme via themeId. Example scenario: "Cross-border disinformation surge before EP elections". Scenarios can exist without a theme link (themeId is nullable) for exploratory work.

  3. 3

    Create Indicators and link them to Scenarios

    Indicators are the measurable tripwires of each scenario. The analyst assigns a strength (1–9) — their prior weighting of how much this indicator matters. Indicators join to scenarios via the scenarioIndicatorMap join table, which carries a separate weight (0.1–10) expressing scenario-level relevance. One indicator can serve multiple scenarios.

  4. 4

    Configure thresholds and decay

    Each indicator has a Watch threshold (default 1.5×) and a Trigger threshold (default 2.5×). timeWeight sets the relevant time window (day / week / month / year). decayBehaviour controls how old signal evidence loses relevance — linear fade, step-function drop, or no decay. Status progresses automatically: normal → watching → triggered.

How scenario warmth is calculated automatically once signals flow in
ℹ
The warmth delta answers "is this scenario heating up or cooling down?" and requires zero analyst effort once indicators are set up. It surfaces on the dashboard as a ranked list, hottest scenarios first.
// ── RECENT WINDOW: last 30 days, linearly decayed ────────────────────────── recentWarmth = Σ ( indicator.strength × recency_weight ) where signalEvent.status = 'approved' AND signalEvent.createdAt ≥ NOW() − 30 days recency_weight = epoch_secs(event_age_in_window) / (30 × 86400) → event from today = weight 1.0, event from 30 days ago = weight ≈ 0 // ── BASELINE WINDOW: 30–60 days ago, no decay ────────────────────────────── prevWarmth = Σ ( indicator.strength ) where signalEvent.status = 'approved' AND signalEvent.createdAt BETWEEN NOW()−60days AND NOW()−30days delta = recentWarmth − prevWarmth δ > 0 → heating up (recent signals outweigh historical baseline) δ < 0 → cooling down (activity lower or weaker than baseline) δ = 0 → stable
⚠
Only approved signal events count. When an analyst approves a signal they are making a direct quantitative contribution to the scenario's warmth metric — not merely filing evidence.

Analyst evidence capture — two paths to linking snippets to indicators

Snippets are analyst-curated evidence fragments — article excerpts, interview quotes, official reports. Linking a snippet to an indicator is what moves curated human evidence into the synopsis engine. Snippets linked only to a theme (the legacy path) provide context but don't participate in per-indicator evidence chains.

Manual Path
  1. Analyst reads source article or report
  2. Creates snippet with content + sourceUrl + pubDate
  3. Pastes a verbatim quote (the key extract)
  4. Selects indicatorId from search-by-name dropdown
  5. Optionally adds analystNotes for reasoning

The Synopsis Engine — Living Dossier Generation

The Theme Synopsis is a structured, citation-backed AI intelligence product. It is triggered on demand (or can be automated). The system knows exactly when the underlying evidence has changed and will signal that the synopsis is stale — no silent drift.

5-step generation pipeline
  1. 1

    assembleThemeContext(themeId)

    Fetches the complete intelligence picture in a single pass: theme metadata → all linked scenarios → all indicators for those scenarios (via scenarioIndicatorMap, fields: id, name, strength, decayBehaviour, status) → top 5 most recent snippets per indicator ordered by pubDate DESC → up to 300 legacy theme-only snippets as background context.

  2. 2

    computeContextHash()

    SHA-256 over a stable string built from: theme (id, name, description) + scenarios (sorted by id) + their indicators (sorted: id, strength, decay, status) + snippet ids + pubDates. If the hash matches themes.synopsisContextHash, generation is skipped entirely — zero unnecessary API spend. Hash changes when any indicator status changes, new snippets are added, or scenario descriptions are updated.

  3. 3

    buildPromptPayload(ctx)

    Clips each snippet to 800 characters. Caps at 100 total snippets, newest first. Formats the context as a structured JSON payload alongside explicit per-field instructions. Indicator strength, decay, and status are included in the payload so the AI can weigh evidence accordingly.

  4. 4

    OpenAI call (gpt-4.1-mini)

    System prompt: "You are an analyst writing a living dossier. Use ONLY the provided snippets as evidence. Return STRICT JSON — no markdown. If evidence is weak or conflicting, say so and surface it in openQuestions." Every claim must be backed by cited snippet IDs. Model cannot invent facts.

  5. 5

    Store + version

    Updates themes.synopsis (JSON blob), increments synopsisVersion, records synopsisModel and the new synopsisContextHash. The UI can compare the live context hash against the stored one at any time to show a "Regenerate" prompt when evidence has drifted.

How indicators directly shape the synopsis
Direct data

Status & strength sent to the AI

Each indicator's strength (1–9), status (normal / watching / triggered), and decayBehaviour are included in the prompt payload. A triggered high-strength indicator is explicitly present in the context — the AI is instructed to assess whether current signals support or undermine each scenario, so indicator state directly drives the scenarioGuidance output section.

Via snippets

Indicator-linked snippets = grounded evidence

The top 5 most recent snippets per indicator arrive as recentSnippets[] on each scenario in the prompt. Every claim in the synopsis must cite snippet IDs — so the entire indicator evidence chain is traceable and auditable from the final output back to the source article, through the analyst quote, to the indicator it was filed against.

AI instruction

ScenarioGuidance output field

The AI is explicitly instructed: "Assess whether current signals support or undermine each scenario." Combined with indicator status, this translates the mechanical warmth delta into natural-language probability guidance — which futures are gaining credibility, which are fading, and why.

Synopsis JSON output schema

synopsis

1–3 paragraph narrative summary of the theme's current state. Can only draw on provided snippets as evidence. No AI hallucination — if evidence is thin the synopsis must say so.

keyPoints [ ]

Bulleted findings. Each item includes citeSnippetIds[]. Every assertion is traceable to a specific analyst-curated snippet and ultimately to a source URL.

openQuestions [ ]

Unanswered questions and conflicting evidence. Each item includes citeSnippetIds[]. This is the system surfacing uncertainty explicitly rather than papering over gaps.

timeline [ ] optional

Time-anchored events: {when, what, citeSnippetIds[]}. Builds a chronological spine for the dossier — essential for trend analysis across a theme.

ℹ
Staleness detection: The UI calls getCurrentContextHash(themeId) and compares it against themes.synopsisContextHash. If they differ — a new snippet was linked, an indicator was triggered, a scenario description changed — the UI shows a "Context has changed — regenerate?" prompt. The synopsis is never silently out-of-date.
How Horizon Lite is an analyst-driven autonomous early warning system

The system divides labour precisely. Automation handles volume and detection. Analysts handle judgment and meaning. Neither can function without the other — removing the automation leaves the analyst drowning in noise; removing the analyst leaves a system that cannot distinguish signal from artefact.

⚙ Automated

GDELT Ingest (hourly cron)

Zero analyst involvement. Raw events, mentions, and document themes are continuously pulled from GDELT's global feed and stored. The system watches 100+ languages around the clock — even when no analyst is logged in. An analyst arriving Monday morning finds three days of evidence already waiting.

⚙ Automated

Signal Generation (hourly)

Four algorithmic detectors scan for theme spikes, tone collapses, amplification anomalies, and geographic hotspots. Each match creates a pending signal event automatically linked to the relevant indicator via a simple text-match routing table. Deduplication prevents the same alert firing repeatedly in the same hourly bucket.

👤 Analyst

Signal Triage — the critical gate

The analyst reviews pending signals and approves or dismisses each one. This is the irreplaceable human-in-the-loop step. An approved signal has been validated by a trained observer — it is not just an algorithm firing on noise. This distinction is what separates Horizon from a news aggregator.

⚙ Automated

Warmth Delta Dashboard

Once signals are approved, the warmth delta for every scenario recalculates automatically using the 30/60-day windowed formula. The dashboard surfaces a ranked list of scenarios sorted by Δ — the hottest futures bubble to the top without any manual scoring or ranking by the analyst. The analyst sees what is heating up before they open a single article.

⚡ AI-Assisted

Snippet Capture + Indicator Linking

Analysts curate evidence snippets from sources they choose. OpenAI can suggest which indicator a quoted passage belongs to, but the analyst always confirms or overrides. The AI suggestion handles the cognitive load of mapping text to taxonomy; the analyst provides domain judgment and catches misrouted suggestions.

👤 + 🤖 On demand

Synopsis Regeneration

Triggered on analyst demand (or can be automated as a scheduled job). The analyst decides when accumulated evidence justifies a new dossier update. The context hash tells them objectively whether anything has changed since the last synopsis — removing the guesswork of "is my dossier current?"

North-star goal: 20 scenarios across 5 themes → Sentinel Assessment in under 5 minutes

GDELT runs 24/7 automatically. Warmth deltas require zero analyst effort. Signal triage takes seconds per item. Snippet linking (with AI assist) takes a minute per source. The synopsis is one button press producing a fully-cited intelligence product. An analyst spends their time on interpretation and judgment — not on information gathering or scoring. The machine handles volume; the analyst handles meaning.
