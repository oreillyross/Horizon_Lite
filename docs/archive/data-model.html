Many-to-many: scenario_indicator_map
An Indicator has a primary owner Scenario via indicators.scenario_id, but it can also be shared across multiple Scenarios through the scenario_indicator_map join table (which carries a per-scenario weight). This means one Indicator — say "Suspicious port intrusions in Baltic states" — can simultaneously contribute evidence to both a "Cyber Escalation" and a "Hybrid Warfare" Scenario, each with a different analyst-assigned weight.

Snippet Capture Workflow

Analyst captures content

Via the web-cut tool or manual entry. A snippet row is created with content, quote, source_url, and pub_date.

Tag to a Theme

The analyst (or the capture UI) sets theme_id. This is a loose link — it just files the snippet under the right domain. The snippet is not yet contributing to any measurement.

AI suggests an Indicator

The server-side AI pipeline reads the snippet and writes its best guess into ai_suggested_indicator_id. No value is changed yet.

Analyst accepts or overrides

The analyst reviews the suggestion. They set indicator_id — either confirming the AI's pick or choosing a different one. They can also add analyst_notes.

Indicator value updates

Once indicator_id is set, the assigned snippet counts as evidence. The indicator's current_value is recalculated, and its status may advance from normal → watching → triggered.

Scenario probability shifts

Because the Indicator is linked to one or more Scenarios (via scenario_id or scenario_indicator_map), the underlying scenario's evidence weight changes — visible in the dashboard.

Key Distinctions

Snippet vs Signal Event

Snippets are analyst-driven: a human chooses to capture something and decides which Indicator it belongs to.

Signal Events are machine-driven: the GDELT pipeline ingests them automatically and the AI assigns a score. The analyst reviews them but doesn't need to go hunting.

theme_id on a Snippet (loose) vs indicator_id (precise)

theme_id on a snippet is a filing tag — it says "this article is broadly about Theme X." It has no numeric effect.

indicator_id is the operational link — it says "this evidence moves the needle on Indicator Y," which in turn affects the parent Scenario's assessment.

indicators.scenario_id vs scenario_indicator_map

indicators.scenario_id is the primary owner: the Scenario this Indicator was originally created for.

scenario_indicator_map enables sharing: the same Indicator can contribute weighted evidence to additional Scenarios without duplicating data.

Indicator status lifecycle

normal — current_value is below threshold_watch. Business as usual.

watching — current_value ≥ threshold_watch (1.5× baseline). Pay attention.

triggered — current_value ≥ threshold_trigger (2.5× baseline). Escalate to the Sentinel Assessment.
